# 《桂子山校园秘境探险》开发计划

> **开发周期**：14天（1-2周冲刺）
> **开发模式**：独立开发
> **技术栈**：Python 3 + Pygame + pygame_gui + pytmx
> **地图工具**：Tiled Map Editor
> **目标**：完整实现游戏设计文档中的所有功能

---

## 一、开发约束与策略

### 1.1 核心约束

| 约束 | 说明 |
|------|------|
| 时间 | 仅14天，每天需高效产出8-10小时 |
| 人力 | 独立开发，无法并行 |
| 美术 | 无专业美术支持，需用像素画工具快速产出 |
| 经验 | 大作业级别，需保证代码可运行、可演示 |

### 1.2 核心开发策略

1. **自底向上 + 增量迭代**：先搭建底层框架，再逐层叠加功能，每一步都可运行测试
2. **先跑通再打磨**：每个模块先实现最小可用版本，再逐步完善细节
3. **资产并行准备**：代码开发与美术/地图资源制作交替进行，避免阻塞
4. **数据驱动**：对话、道具、谜题数据全部外置为JSON，减少硬编码，方便调整
5. **复用优先**：UI组件、实体逻辑尽量抽象为通用基类，减少重复代码
6. **UI三阶段递进**：UI先实现基础功能（纯色/矩形占位），再替换为像素风素材，最后精细打磨动画与主题效果

### 1.3 关键路径（Critical Path）

以下模块存在严格依赖关系，任何延迟都会阻塞后续开发：

```
Pygame窗口框架 → Tile地图渲染 → 玩家移动+碰撞 → 摄像机跟随
    → 互动系统 → NPC对话 → 背包系统 → 谜题系统 → 完整游戏流程
```

**地图资源制作是另一条关键路径**：
```
学习Tiled → 绘制本部校区地图 → 绘制室内地图 → 绘制南湖校区地图
```

两条路径需并行推进。

**光照实现方案**：
- 采用关键帧插值法：全天11个关键帧，任意时刻颜色由相邻帧线性插值得到
- 白天(8:00-16:00)：无色调叠加，正午最亮
- 清晨(6:00-8:00)：暖橘色渐出，模拟日出
- 下午(16:00-18:00)：微暖色渐入，模拟傍晚
- 黄昏(18:00-19:30)：橘红色→紫蓝色过渡，无硬切换
- 夜晚(19:30-6:00)：深蓝色叠加(alpha最高155)，色调(10,15,55)
- 凌晨(3:00-6:00)：深蓝渐亮→暖紫→日出暖光，自然过渡到白天
- 秘境蛰伏态(白天)：极淡暖绿色叠加(alpha≈8-12)，校园微妙异常
- 秘境觉醒态(夜晚)：深蓝叠加+幽绿色脉冲(alpha 20-50呼吸)，路灯发绿光

### 4.3 `world/tilemap.py` — 地图渲染

**核心功能**：
1. 通过 `pytmx.load_pygame()` 加载TMX文件
2. 按层渲染：地面层 → 建筑层 → 对象层 → 碰撞层（不渲染）
3. 视口裁剪：计算摄像机可见的Tile范围，仅渲染该范围
4. 碰撞数据：从Tiled的碰撞属性中提取，生成二维布尔数组

**地图数据结构**：
```python
class TileMap:
    tmx_data: pytmx.TiledMap       # Tiled地图数据
    collision_map: list[list[bool]] # 碰撞二维数组
    width: int                      # 地图宽度（tiles）
    height: int                     # 地图高度（tiles）
    spawn_points: dict              # 各场景出生点坐标
    transitions: list               # 过渡触发区域列表
```

## 五、UI设计规格

### 5.1 UI技术架构

采用三层架构，将交互逻辑与视觉渲染分离：

```
┌─────────────────────────────────────────┐
│           游戏主循环 (Pygame)             │
│                                         │
│  ┌─────────────┐  ┌──────────────────┐  │
│  │  pygame_gui  │  │  自定义渲染层     │  │
│  │  (布局/交互) │  │  (像素风外观)     │  │
│  │  - 按钮事件  │  │  - 9-slice边框   │  │
│  │  - 窗口管理  │  │  - 像素图标      │  │
│  │  - 文字排版  │  │  - 动画效果      │  │
│  └─────────────┘  └──────────────────┘  │
│                                         │
│  ┌─────────────────────────────────────┐│
│  │        自定义主题 (theme.json)       ││
│  │  - 像素风字体                        ││
│  │  - 颜色/间距配置                     ││
│  │  - 按钮图片路径                      ││
│  └─────────────────────────────────────┘│
└─────────────────────────────────────────┘
```

**核心思路**：pygame_gui 负责交互逻辑（点击、悬停、焦点、滚动、窗口管理），外观通过自定义主题覆盖为像素风。部分 pygame_gui 无法实现的视觉效果（如9-slice边框），在自定义渲染层中用纯 Pygame 绘制覆盖。

**pygame_gui 与自定义渲染的分工**：

| 职责 | pygame_gui 处理 | 自定义渲染层处理 |
|------|----------------|----------------|
| 布局与定位 | ✅ 所有UI元素的尺寸和位置 | — |
| 交互事件 | ✅ 点击、悬停、焦点、滚动 | — |
| 文字排版 | ✅ 字体渲染、换行、对齐 | — |
| 边框外观 | — | ✅ 9-slice像素风边框 |
| 图标绘制 | — | ✅ 道具图标、HUD图标 |
| 动画效果 | — | ✅ 体力条渐变、徽章发光等 |

### 5.2 9-slice边框技术

9-slice（又称9-patch）是像素风UI的核心技术——将一个UI框图片分为9个区域，拉伸时只拉伸边和中心，角保持不变形，保证像素风边框在任何尺寸下都不走样。

```
┌───┬───────────────┬───┐
│ 1 │      2        │ 3 │   1,3,7,9 = 角（不拉伸）
├───┼───────────────┼───┤   2,4,6,8 = 边（单方向拉伸）
│ 4 │      5        │ 6 │   5 = 中心（双向拉伸）
├───┼───────────────┼───┤
│ 7 │      8        │ 9 │
└───┴───────────────┴───┘
```

**pygame_gui 兼容性问题**：pygame_gui 的主题系统支持为元素指定背景图片，但不支持9-slice分割参数——加载的图片会被整体拉伸到元素尺寸，导致像素风边框的四个角变形。因此需要备选方案。

**备选方案：分层渲染**

1. pygame_gui 元素设置 `border_width=0`、`shadow_width=0`，背景色设为透明
2. 在 pygame_gui 渲染之前，用自写的9-slice渲染函数在目标位置绘制像素风边框背景
3. pygame_gui 只负责绘制文字和交互状态

这种方式不侵入 pygame_gui 的交互逻辑，约需额外50-80行渲染代码。

**实现策略**：阶段1先用 pygame_gui 的纯色边框（不使用图片），阶段2再切换为分层渲染+9-slice，避免阶段1就引入复杂度。

### 5.3 像素风字体方案

字体对UI美观度影响极大。必须使用点阵字体（而非矢量字体），并确保字体大小与像素网格对齐，否则中文渲染会模糊失真。

| 字体 | 风格 | 许可证 | 说明 |
|------|------|--------|------|
| **Zpix（首选）** | 中文像素风 | 免费 | 专为中文像素游戏设计，强烈推荐 |
| Fusion Pixel（备选） | 中文像素风 | 免费 | 完美支持中文，多种字重 |
| DotGothic16（备选） | 日系像素风 | 免费 | 支持CJK，Google Fonts可获取 |
| Press Start 2P | 英文像素风 | 免费 | 经典8-bit风格，仅英文 |

**字体使用要点**：
- 主字体选用 Zpix，在 theme.json 的 `defaults.font` 中配置
- 字体大小必须是像素网格的整数倍（推荐12px、16px），避免亚像素渲染导致模糊
- 英文部分（如按键提示"E"）可用 Press Start 2P 作为补充

**临时字体方案（阶段1）**：

Zpix 字体计划在阶段2（Day 12-13）正式配置。但阶段1（Day 1-11）已有中文文本渲染需求（标题画面、暂停菜单、HUD 等），因此采用临时方案：

| 阶段 | 字体 | 加载方式 | 说明 |
|------|------|---------|------|
| 阶段1（临时） | 系统黑体（SimHei） | `pygame.font.Font(FONT_PATH, size)` | 矢量字体，非像素风，仅保证中文可显示 |
| 阶段2（正式） | Zpix | `pygame.font.Font(ZPIX_PATH, size)` + theme.json | 像素风字体，与游戏风格统一 |

- `FONT_PATH` 定义在 `config.py` 中，值为系统黑体路径（如 `C:/Windows/Fonts/simhei.ttf`）
- 临时字体仅保证中文**可显示**，不保证美观（矢量字体在低分辨率下会模糊）
- 阶段2切换 Zpix 后，删除 `FONT_PATH` 常量，所有字体加载统一使用 Zpix

### 5.4 美术素材获取策略

美术素材是UI美观度的核心瓶颈。采用三条路径组合获取：

**路径一：免费像素风素材包（推荐首选）**

| 素材来源 | 内容 | 许可证 | 说明 |
|----------|------|--------|------|
| itch.io（免费像素素材区） | UI框、图标、按钮 | 多为CC0或CC-BY | 搜索 "pixel art UI pack free"，有大量免费素材 |
| opengameart.org | UI框架、物品图标 | CC0/GPL | 老牌游戏素材站，质量参差需筛选 |
| kenney.nl | 完整UI套件 | CC0 | 风格偏简约，但统一性好，适合做基础 |
| 0x72.itch.io（Dungeontileset） | 像素风UI+物品图标 | CC0 | 高质量像素风，社区口碑极好 |

**推荐组合**：
- **UI边框/面板/按钮**：从 itch.io 找一套像素风 RPG UI pack（搜索 "RPG UI pixel"），确保包含9-slice友好的PNG
- **道具图标（通用类）**：从 0x72 或 kenney.nl 的物品图标包中选取
- **角色头像**：用路径三（AI生成）
- **校园特色元素**：用路径三（AI生成）

**路径二：程序化生成像素图标**

适用于几何形状简单的道具（钥匙、石头、书本、徽章等），用 Pygame 代码直接绘制。不适用于复杂角色头像和精细装饰图案。

**路径三：AI辅助生成像素画**

适用于角色头像、校园特色道具（桂花徽章、旧校徽、桂花枝等）、UI装饰元素（桂花纹样等）。AI生成后用 Piskel（免费在线）或 LibreSprite（免费开源）手动微调，建立统一调色板。

**素材风格统一原则**：选定一个主素材包作为视觉基准，其他来源的素材需手动调色统一，避免UI元素风格割裂。

### 6.3 Tiled地图编辑规范

   - `structures`：建筑/墙壁层

   - 可互动物体需添加 `interactive_type` 属性（dialog/examine/pickup/use/mechanism）
   - 门口需添加 `target_map` 和 `spawn_point` 属性
   - NPC需添加 `npc_id` 属性
   - 碰撞Tile需添加 `solid=true` 属性

## 八、风险与应对

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|----------|
| 时间不足，无法完成全部功能 | 高 | 高 | 优先保证核心流程可通关，昼夜/UI/音效可简化 |
| Tiled地图制作耗时超预期 | 高 | 中 | 先用简化版地图（减少装饰细节），后期再打磨 |
| 像素精灵制作耗时超预期 | 中 | 高 | 先用色块/简单图形占位，后期替换；或使用免费素材 |
| pytmx兼容性问题 | 中 | 低 | 提前测试pytmx加载，备选方案为手动解析TMX的XML |
| 谜题逻辑Bug | 中 | 中 | 每个谜题独立测试，状态机严格管理 |
| 性能问题（大地图卡顿） | 中 | 中 | 严格实现视口裁剪，必要时分Chunk加载 |
| 存档数据损坏 | 低 | 低 | 存档时写临时文件再重命名，避免写入中断 |
| UI素材风格不统一 | 中 | 中 | 选定一个主素材包为基准，其他素材手动调色统一 |
| pygame_gui 9-slice支持不完善 | 中 | 低 | 必要时自写9-slice渲染函数，采用分层渲染绕过pygame_gui图片渲染 |
| 中文字体渲染模糊 | 中 | 中 | 使用点阵字体(Zpix)而非矢量字体，字体大小与像素网格对齐 |

## 十一、代码规范

### 11.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名 | snake_case | `game_manager.py` |
| 类名 | PascalCase | `GameManager` |
| 函数/方法 | snake_case | `load_map()` |
| 常量 | UPPER_SNAKE_CASE | `TILE_SIZE` |
| 私有属性 | _前缀 | `_internal_state` |

### 11.2 代码结构规范

- 每个类一个文件（简单辅助类可合并）
- 模块间通过接口通信，避免循环导入
- 游戏逻辑与渲染分离（update/draw 分离）
- 数据与逻辑分离（JSON数据文件 vs Python代码）

### 11.3 注释规范

- 文件顶部：模块用途说明
- 类：职责说明
- 复杂方法：关键步骤注释
- 魔法数字：必须注释含义
