# 《桂子山校园秘境探险》开发计划

> **开发周期**：14天（1-2周冲刺）
> **开发模式**：独立开发
> **技术栈**：Python 3 + Pygame + pytmx
> **地图工具**：Tiled Map Editor
> **目标**：完整实现游戏设计文档中的所有功能

---

## 一、开发约束与策略

### 1.1 核心约束

| 约束 | 说明 |
|------|------|
| 时间 | 仅14天，每天需高效产出8-10小时 |
| 人力 | 独立开发，无法并行 |
| 美术 | 无专业美术支持，需用像素画工具快速产出 |
| 经验 | 大作业级别，需保证代码可运行、可演示 |

### 1.2 核心开发策略

1. **自底向上 + 增量迭代**：先搭建底层框架，再逐层叠加功能，每一步都可运行测试
2. **先跑通再打磨**：每个模块先实现最小可用版本，再逐步完善细节
3. **资产并行准备**：代码开发与美术/地图资源制作交替进行，避免阻塞
4. **数据驱动**：对话、道具、谜题数据全部外置为JSON，减少硬编码，方便调整
5. **复用优先**：UI组件、实体逻辑尽量抽象为通用基类，减少重复代码

### 1.3 关键路径（Critical Path）

以下模块存在严格依赖关系，任何延迟都会阻塞后续开发：

```
Pygame窗口框架 → Tile地图渲染 → 玩家移动+碰撞 → 摄像机跟随
    → 互动系统 → NPC对话 → 背包系统 → 谜题系统 → 完整游戏流程
```

**地图资源制作是另一条关键路径**：
```
学习Tiled → 绘制本部校区地图 → 绘制室内地图 → 绘制南湖校区地图
```

两条路径需并行推进。

**状态转换**：
```
TITLE ──开始──→ INTRO ──完成──→ PLAYING
```

### 4.3 `world/tilemap.py` — 地图渲染

**核心功能**：
1. 通过 `pytmx.load_pygame()` 加载TMX文件
2. 按层渲染：地面层 → 建筑层 → 对象层 → 碰撞层（不渲染）
3. 视口裁剪：计算摄像机可见的Tile范围，仅渲染该范围
4. 碰撞数据：从Tiled的碰撞属性中提取，生成二维布尔数组

**地图数据结构**：
```python
class TileMap:
    tmx_data: pytmx.TiledMap       # Tiled地图数据
    collision_map: list[list[bool]] # 碰撞二维数组
    width: int                      # 地图宽度（tiles）
    height: int                     # 地图高度（tiles）
    spawn_points: dict              # 各场景出生点坐标
    transitions: list               # 过渡触发区域列表
```

### 5.3 Tiled地图编辑规范

1. **图层命名规范**：
   - `ground`：地面层（草地、路面、地板）
   - `terrain`：地形装饰层（花坛边缘、路面纹理）
   - `structures`：建筑/墙壁层
   - `objects`：可互动物体层（Tiled对象层，非Tile层）
   - `collision`：碰撞层（不可见，仅标记碰撞）
   - `triggers`：触发区域层（门口、楼梯、过渡点）
   - `decorations`：装饰层（路灯、长椅等纯装饰）

2. **自定义属性**：
   - 可互动物体需添加 `interactive_type` 属性（dialog/examine/pickup/use/mechanism）
   - 门口需添加 `target_map` 和 `spawn_point` 属性
   - NPC需添加 `npc_id` 属性
   - 碰撞Tile需添加 `solid=true` 属性

## 十、代码规范

### 10.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名 | snake_case | `game_manager.py` |
| 类名 | PascalCase | `GameManager` |
| 函数/方法 | snake_case | `load_map()` |
| 常量 | UPPER_SNAKE_CASE | `TILE_SIZE` |
| 私有属性 | _前缀 | `_internal_state` |

### 10.2 代码结构规范

- 每个类一个文件（简单辅助类可合并）
- 模块间通过接口通信，避免循环导入
- 游戏逻辑与渲染分离（update/draw 分离）
- 数据与逻辑分离（JSON数据文件 vs Python代码）

### 10.3 注释规范

- 文件顶部：模块用途说明
- 类：职责说明
- 复杂方法：关键步骤注释
- 魔法数字：必须注释含义
