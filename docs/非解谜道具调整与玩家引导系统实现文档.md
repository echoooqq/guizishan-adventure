### 注意事项

1. **坐标微调**：文档中的坐标基于当前TMX地图的trigger坐标推算，实际放置时需要进入游戏验证位置是否合理，避免与碰撞层重叠或被墙壁遮挡。

2. **NPC与碰撞**：NPC没有碰撞体积（玩家可以直接穿过），因此不需要担心NPC阻挡路径。

3. `_setup_test_entities()` **方法膨胀**：此方法已包含大量实体创建代码，新增6个main_campus地图的引导NPC后会进一步膨胀。建议后续重构时将main_campus的实体创建拆分为多个子方法（如 `_setup_gate_entities()`、`_setup_library_entrance_entities()` 等），但本次实现暂不重构。

4. **精灵图降级**：如果精灵图生成失败，NPC会使用代码绘制的矩形拼色块，不影响游戏功能。可以后续再补充精灵图。

5. **教程兼容**：水壶位置微调后，路过的学生对话中提到"那边地上有个水壶"仍然适用，因为水壶仍在出生点附近。教程步骤 `TUTORIAL_PICKUP` 检测的是 `item_id == "water_bottle"` 的拾取事件，与位置无关。

6. **道具拾取与存档**：拾取物使用 `InteractiveObject.interacted` 标记是否已拾取，但此标记不直接存档。存档恢复时，`_load_map()` 会重新调用各 setup 方法重建所有实体。对于已拾取的道具，需要在 setup 方法中检查背包是否已有该道具来避免重复生成。**当前代码未做此检查**，这意味着如果玩家拾取了桂花枝后存档再读档，桂花枝会重新出现在地图上。这是一个已有的问题，不在本次修改范围内，但需要注意。

7. **main_campus地图的NPC数量**：新增6个引导NPC后，main_campus地图上将有8个NPC（路过的学生、神秘学长、晨读学姐、还书同学、写生同学、运动同学、等餐同学、等车同学）+ 1个守护者（条件可见）+ 1个广场舞阿姨（夜晚可见）。需注意性能影响，但当前NPC逻辑简单（仅待机动画+可见性判断），不会有明显性能问题。

8. **神秘学长位置调整**：当前神秘学长在 main_campus 出生点左侧48像素处（第308-318行），与路过的学生相邻。按照游戏逻辑，神秘学长应该只在南湖综合楼1楼出现，不应出现在本部校区出生点。但此调整不在本次方案范围内，记录待后续处理。
