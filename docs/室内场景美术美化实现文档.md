# 室内场景美术美化与统一 — 详细实现文档

> 本文档基于项目当前代码实际实现编写，所有改动均通过修改离线生成工具代码实现，不涉及运行时游戏代码。
> 后续按本文档逐步落实，每步完成后运行对应生成工具验证效果。

---

## 一、改动范围总览

### 1.1 需要修改的文件

| 文件 | 改动类型 | 说明 |
|------|---------|------|
| `tools/generate_indoor_maps.py` | **主要修改** | 重绘低质量 tile、新增 tile、修复 GID 冲突、统一透明背景、更新地图布局 |
| `tools/generate_sprites.py` | **次要修改** | 重绘低质量精灵（dining_table、computer_terminal、floor_lamp、magazine_rack、water_dispenser） |
| `tools/generate_nanhu_maps.py` | **轻微修改** | 更新综合楼/密室室内地图布局，引用新增 tile |
| `entities/interactive_object.py` | **可能修改** | 如新增精灵映射需更新 `OBJECT_SPRITE_MAP` |
| `world/tilemap.py` | **可能修改** | 如新增 `_TYPE_SPRITE_MAP` 映射需更新 |

### 1.2 不需要修改的文件

- `main.py`、`config.py`、`game/game_manager.py` — 运行时逻辑不变
- `world/tilemap.py` — 地图加载/渲染逻辑不变（仅可能新增精灵映射）
- 室外地图生成工具 — 不涉及

### 1.3 输出产物

每次修改生成工具后，运行工具重新生成以下文件：

```bash
python tools/generate_indoor_maps.py    # → assets/tilesets/indoor_tileset.png + world/map_data/*.tmx
python tools/generate_sprites.py         # → assets/sprites/*.png
python tools/generate_nanhu_maps.py      # → world/map_data/nanhu_campus.tmx + nanhulou_*.tmx
```

## 三、阶段 A：基础修复

### 步骤 A1：修复 GID 冲突

**问题**：`GID_HOOP_BOT = 21` 与 `GID_KITCHEN_FLOOR = 21` 使用相同 GID，后者覆盖前者绘制。

**修改位置**：`tools/generate_indoor_maps.py` 第 32 行

**具体操作**：
1. 将 `GID_KITCHEN_FLOOR = 21` 改为 `GID_KITCHEN_FLOOR = 70`
2. 在 `_draw_tile()` 函数中，将 `elif gid == GID_KITCHEN_FLOOR:` 分支的绘制代码移到文件末尾（GID 70 的位置）
3. 更新 `TILE_COUNT = 71`（从 70 改为 71）
4. 在 `SOLID_GIDS` 集合中确认 `GID_KITCHEN_FLOOR` 的新值已正确引用

**验证**：运行 `python tools/generate_indoor_maps.py`，检查生成的 `indoor_tileset.png`：
- GID 21 位置应为篮球架下部（支柱+底座），不再是厨房地面
- GID 70 位置应为厨房地面（棋盘格瓷砖）

### 步骤 A2：统一家具 tile 透明背景

**问题**：以下 tile 使用地板色 `(180, 140, 90)` 填充背景，导致只能放在木地板上：

| GID | 名称 | 当前背景色 | 修改为 |
|-----|------|-----------|--------|
| 9 | TABLE | `(180, 140, 90)` 木地板色 | `(0, 0, 0, 0)` 透明 |
| 10 | CHAIR | `(180, 140, 90)` 木地板色 | `(0, 0, 0, 0)` 透明 |
| 11 | COUNTER | `(180, 140, 90)` 木地板色 | `(0, 0, 0, 0)` 透明 |
| 12 | FRIDGE | `(180, 140, 90)` 木地板色 | `(0, 0, 0, 0)` 透明 |
| 17 | SCOREBOARD | `(240, 240, 245)` 墙壁色 | `(0, 0, 0, 0)` 透明 |
| 18 | COMPUTER | `(180, 140, 90)` 木地板色 | `(0, 0, 0, 0)` 透明 |
| 22 | RUG | `(180, 140, 90)` 木地板色 | `(0, 0, 0, 0)` 透明 |
| 23 | BLACKBOARD | `(180, 140, 90)` 木地板色 | `(0, 0, 0, 0)` 透明 |
| 31 | GYM_LOCKER | `(240, 240, 245)` 墙壁色 | `(0, 0, 0, 0)` 透明 |
| 28 | LIB_READING_LAMP | `(120, 80, 45)` 地板色 | `(0, 0, 0, 0)` 透明 |

**具体操作**：在每个 tile 的 `_draw_tile()` 分支中，将 `surface.fill(背景色, rect)` 改为 `surface.fill((0, 0, 0, 0), rect)`。

**注意**：改为透明背景后，原来依赖背景色填充来"遮盖"地面 tile 的效果将消失。对于桌子、椅子等家具，这是正确行为——家具应该叠加在地面上。但需要注意：
- `RUG`（地毯）改为透明后，需要自己绘制完整的地毯外观（包含边缘），不能依赖背景色
- `BLACKBOARD` 改为透明后，木框需要自行绘制完整，不能依赖背景色填充

**验证**：运行生成工具后，在 Tiled Map Editor 中打开各室内地图，检查家具 tile 是否能正确叠加在不同地面色上，不出现色块。

### 步骤 A3：统一阴影规范

**规范**：所有透明背景的家具/装饰物 tile，底部增加 1px 深色阴影线，颜色为 `(地面色 RGB 各减 40, alpha=80)` 的半透明像素。

**具体操作**：在 `_draw_tile()` 中，对每个透明背景的家具 tile，在绘制完主体后，在物体底部 y 坐标处绘制 1 行半透明阴影像素。使用 `pygame.SRCALPHA` Surface 实现。

**示例代码模式**：
```python
# 底部阴影（1px 高，半透明深色）
shadow = pygame.Surface((物体宽度, 1), pygame.SRCALPHA)
shadow.fill((0, 0, 0, 40))
surface.blit(shadow, (x + 偏移, 物体底部y))
```

**适用 tile**：TABLE、CHAIR、COUNTER、FRIDGE、COMPUTER、BLACKBOARD、GYM_LOCKER、BOOKSHELF、BOOKSHELF_TOP、OFFICE_DESK、LARGE_DESK_L/R、所有体育馆设施 tile、POTTED_PLANT、SOFA、DISPLAY_CABINET、SECTION_SIGN

**不适用 tile**：地面类（WOOD_FLOOR、TILE_FLOOR 等）、墙壁类（WALL 系列）、碰撞标记（COLLISION）

- 右椅腿：x=10, y=10~13, 深棕 (90, 55, 25)

## 九、关于素材参考

### 9.1 是否需要下载素材包？

**不需要**。本项目所有美术资源均通过 `pygame.draw` 系列 API 程序化生成，不使用外部素材包。这种方式的优点是：
- 完全可控，修改代码即可调整
- 不存在版权问题
- 风格统一性由代码保证

### 9.2 是否需要上网找参考？

**建议参考但非必须**。如需提升绘制质量，可搜索以下参考：
- "16x16 pixel art furniture" — 了解像素风家具的标准画法
- "pixel art top down desk chair" — 俯视角桌椅的常见表现方式
- "pixel art cafeteria interior" — 食堂室内场景参考
- "pixel art library interior" — 图书馆室内场景参考

参考目的：确认比例、颜色搭配、细节层次，而非照搬。

### 9.3 绘制质量标杆

本项目内已有的高质量绘制可作为标杆：

| 标杆物体 | 文件 | 值得学习的特征 |
