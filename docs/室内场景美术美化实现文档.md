## 二、阶段1：修复 GID 冲突 + 统一透明背景

### 2.1 修复 GID 冲突

**问题**：`GID_HOOP_BOT = 21` 和 `GID_KITCHEN_FLOOR = 21` 使用了相同的 GID 值。在 `_draw_tile()` 函数中，`elif gid == GID_KITCHEN_FLOOR`（第353行）位于 `elif gid == GID_HOOP_BOT`（第313行）之后，但由于两者 GID 值相同，`HOOP_BOT` 的 elif 分支会先被匹配到，`KITCHEN_FLOOR` 的分支永远不会执行。

**修复方案**：

1. 在 `generate_indoor_maps.py` 第87行后新增常量：
   ```python
   GID_KITCHEN_FLOOR = 70   # 原 GID=21 与 HOOP_BOT 冲突，迁移到新编号
   ```

2. 删除第32行的旧定义：
   ```python
   # 删除：GID_KITCHEN_FLOOR = 21
   ```

3. 更新 `TILE_COUNT`：
   ```python
   TILE_COUNT = 71  # 原 70，新增1个
   ```

4. 更新 `SOLID_GIDS` 集合：确认 `GID_KITCHEN_FLOOR` 不在集合中（当前确实不在，无需修改）。

5. 搜索所有使用 `GID_KITCHEN_FLOOR` 的位置，确认只有以下两处：
   - `design_dining_hall_f2()` 第1793-1796行：`ground[2][x] = GID_KITCHEN_FLOOR` 等
   - `_draw_tile()` 第353行：`elif gid == GID_KITCHEN_FLOOR`

   这些引用使用的是常量名而非硬编码数字，修改常量值即可自动生效。

6. **同步更新 `generate_nanhu_maps.py`**：该文件第653行也定义了 `GID_KITCHEN_FLOOR = 21`，需同步改为 `GID_KITCHEN_FLOOR = 70`，并更新 `TILE_COUNT_I = 71`。

### 2.2 统一透明背景

**问题**：以下 tile 使用地板色 `(180, 140, 90)` 或墙壁色填充背景，导致只能放在对应颜色的地面上：

| GID | 名称 | 当前背景色 | 行号 |
|-----|------|-----------|------|
| GID_TABLE (9) | 桌子 | `(180, 140, 90)` 木地板色 | 224 |
| GID_CHAIR (10) | 椅子 | `(180, 140, 90)` 木地板色 | 244 |
| GID_COUNTER (11) | 柜台 | `(180, 140, 90)` 木地板色 | 250 |
| GID_FRIDGE (12) | 冰箱 | `(180, 140, 90)` 木地板色 | 257 |
| GID_COMPUTER (18) | 电脑 | `(180, 140, 90)` 木地板色 | 336 |
| GID_BLACKBOARD (23) | 黑板 | `(180, 140, 90)` 木地板色 | 369 |
| GID_GYM_LOCKER (31) | 更衣柜 | `(240, 240, 245)` 墙壁色 | 430 |
| GID_SCOREBOARD (17) | 记分牌 | `(240, 240, 245)` 墙壁色 | 332 |
| GID_LIB_READING_LAMP (28) | 阅读灯 | `(120, 80, 45)` 地板色 | 409 |
| GID_DOOR_INDOOR (19) | 室内门 | `(180, 140, 90)` 木地板色 | 344 |
| GID_STAIRS_DOWN (13) | 下楼楼梯 | `(180, 140, 90)` 木地板色 | 264 |
| GID_STAIRS_UP (14) | 上楼楼梯 | `(180, 140, 90)` 木地板色 | 272 |
| GID_RUG (22) | 地毯 | `(180, 140, 90)` 木地板色 | 362 |
| GID_CARPET_RED (4) | 红地毯 | `(180, 140, 90)` 木地板色 | 153 |
| GID_COLLISION (20) | 碰撞标记 | `(180, 140, 90)` 木地板色 | 349 |

**修复方案**：

将上述 tile 的 `surface.fill(不透明色, rect)` 改为 `surface.fill((0, 0, 0, 0), rect)`（透明背景）。

**注意事项**：

- **楼梯/门/地毯/碰撞标记**：这些 tile 跨场景通用，改为透明背景后需要确保地面层（ground layer）已正确铺设对应地面 tile。当前所有室内地图的 ground 层已完整铺设（`_fill_layer` 全量填充），因此改为透明不会出现"漏底"问题。
- **碰撞标记 GID_COLLISION**：当前渲染为半透明红色叠加层，仅用于调试。改为透明背景后，红色叠加层仍保留（通过 `overlay.fill((255, 0, 0, 60))` + `blit` 实现），只是底层从木地板色变为透明。
- **阅读灯 GID_LIB_READING_LAMP**：当前使用图书馆地板色 `(120, 80, 45)` 作为背景，改为透明后，暖光光晕效果在透明背景上可能需要调整 alpha 值（当前 alpha=80，透明背景上可能显得过亮，可降低到 50-60）。

**具体修改**：

对每个需要改透明的 tile，将 `_draw_tile()` 函数中的：
```python
surface.fill((180, 140, 90), rect)  # 或其他不透明色
```
替换为：
```python
surface.fill((0, 0, 0, 0), rect)  # 透明背景
```

同时，对于桌子、椅子等家具 tile，原来用背景色填充的区域（如桌子下方的"地面"部分）需要删除，只保留家具本身的像素。
