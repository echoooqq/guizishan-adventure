## 一、最高优先级：游戏界面必须使用中文

**这是本项目最核心的规范，违反此条即视为严重缺陷。**

本项目是一款以华中师范大学校园为背景的中文解谜冒险游戏，目标用户为中文玩家。所有面向玩家的可见文本必须使用中文。

### 1.2 允许使用英文的场景

仅以下场景允许使用英文：

- **按键名称**：`WASD`、`F`、`Esc`、`Shift`、`Tab`、`M`、`I` 等键盘按键标识（这是国际通用标准）
- **代码内部标识符**：变量名、函数名、类名、文件名等编程标识符（遵循 Python 命名规范）
- **JSON 数据键名**：`"npc_id"`、`"puzzle_id"` 等数据结构的键名
- **调试信息**：仅开发阶段使用的控制台日志（但正式版应移除或改为中文）
- **Tiled 地图属性名**：`"interactive_type"`、`"target_map"` 等自定义属性键名

### 1.3 字体规范

- **正式字体**：必须使用 Zpix（中文像素风字体），确保中文渲染清晰
- **字体加载**：`config.py` 中 `FONT_PATH` 优先加载 Zpix（`assets/fonts/zpix.ttf`），若不存在则降级使用系统黑体（SimHei）
- **字体大小**：必须是像素网格的整数倍（推荐 12px、16px），避免亚像素渲染导致中文模糊
- **英文补充字体**：按键提示中的英文按键名可用 Press Start 2P
- **禁止使用** `pygame.font.Font(None, size)` 渲染中文文本，该默认字体不支持中文
- **Zpix字体下载**：运行 `python tools/download_zpix.py` 或手动从 https://github.com/SolidZORO/zpix-pixel-font/releases 下载 `zpix.ttf` 放到 `assets/fonts/`
- **注意**：Zpix 不含特殊符号（如 ▶），需用 `pygame.draw.polygon` 等绘制替代

## 二、项目概述

### 2.1 游戏简介

- **名称**：桂子山秘境探险
- **类型**：2D 俯视角 · 校园探索 · 解谜冒险
- **参考风格**：《星露谷物语》俯视角 + 《塞尔达：缩小帽》场景解谜
- **背景设定**：华中师范大学（华师）校园，玩家扮演误入秘境的新生，需集齐 7 枚桂花徽章碎片解除封印

### 2.2 技术栈

| 技术 | 用途 |
|------|------|
| Python 3 | 主语言 |
| Pygame | 渲染引擎、输入处理 |
| pygame_gui | UI 交互框架（布局、事件、窗口管理） |
| pytmx | 加载 Tiled Map Editor 导出的 TMX 地图 |
| Tiled Map Editor | 地图编辑工具 |
| JSON | 数据驱动（对话、道具、谜题、存档） |

### 2.3 渲染架构

```
内部分辨率 480×270 → 缩放至窗口 960×540
Tile 尺寸 16×16 像素
角色约 16×24 像素
```

所有游戏逻辑在内部分辨率下运算，最终通过 `pygame.transform.scale` 缩放到窗口尺寸。

```
    ↓
第二层：tilemap.py · camera.py · entity.py · collision
    ↓

    ↓
第四层A：UI 系统          第四层B：游戏系统
(pygame_gui, dialog,      (inventory, clock,
 hud, minimap, menu)       save_manager, puzzle_manager, transition)
    ↓                          ↓
第五层：内容层（7 个谜题实现 · 对话数据 · 道具数据 · 地图资源 · 音频资源）
    ↓
第六层：流程层（标题画面 · 开场动画 · 教程 · 结局动画）
```

同层模块之间尽量解耦，通过事件/接口通信，避免循环导入。

## 四、代码规范

### 4.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名 | snake_case | `game_manager.py` |
| 类名 | PascalCase | `GameManager` |
| 函数/方法 | snake_case | `load_map()` |
| 常量 | UPPER_SNAKE_CASE | `TILE_SIZE` |
| 私有属性/方法 | _前缀 | `_internal_state` |
| JSON 数据键 | snake_case | `"npc_id"`, `"puzzle_id"` |

### 4.2 代码结构规范

- 每个类一个文件（简单辅助类可合并）
- 模块间通过接口通信，禁止循环导入
- **游戏逻辑与渲染分离**：`update()` 和 `draw()` 分离
- **数据与逻辑分离**：对话、道具、谜题数据全部外置为 JSON，减少硬编码
- 禁止在代码中硬编码对话文本、道具描述等面向玩家的字符串，应从 JSON 数据文件读取

### 4.3 注释规范

- 文件顶部：模块用途说明
- 类：职责说明
- 复杂方法：关键步骤注释
- 魔法数字：必须注释含义
- 注释语言：使用中文

### 4.4 import 规范

- 标准库 → 第三方库 → 项目内模块，三组之间空行分隔
- 禁止使用 `from module import *`
- 优先使用明确的具体导入

## 六、核心系统实现规范

### 6.1 移动系统

- 移动键：WASD / 方向键
- 冲刺：Shift + 移动，消耗体力
- 互动键：F / 空格
- 碰撞检测：基于 Tile，分轴检测（先 X 后 Y），允许沿墙滑动
- 移动速度：步行 2 tiles/秒，冲刺 3.5 tiles/秒

### 6.2 互动系统

互动提示气泡必须使用中文：

| 互动类型 | 提示文本 |
|----------|---------|
| 对话 | `"按 F 对话"` |
| 查看 | `"按 F 查看"` |
| 拾取 | `"按 F 拾取"` |
| 使用 | `"按 F 使用"` |
| 机关操作 | `"按 F 操作"` |
| 摇树 | `"按 F 摇树"` |

### 6.3 背包系统

- 最大 20 格
- 道具分类：关键道具(key_item)、工具(tool)、线索(clue)、消耗品(consumable)、材料(material)
- 关键道具不可丢弃
- 支持道具组合

背包界面中文标签：

| 元素 | 中文文本 |
|------|---------|
| 面板标题 | `"背包"` |
| 使用按钮 | `"使用"` |
| 丢弃按钮 | `"丢弃"` |
| 组合按钮 | `"组合"` |
| 返回按钮 | `"返回"` |
| 关闭按钮 | `"关闭"` |
| 空格子提示 | `"空"` |

### 6.4 对话系统

- 对话数据存储在 `data/dialogues/` 目录下的 JSON 文件中
- 每个NPC一个JSON文件
- 对话框底部显示，左侧说话者头像+名称
- 文字逐字显示（打字机效果，每字间隔 50ms）
- 按F/空格加速显示或跳到下一句
- 支持选项分支（方向键选择，F确认）
- 所有对话文本必须使用中文

对话数据格式：
```json
{
  "npc_librarian": {
    "default": [
      {"speaker": "图书管理员", "text": "欢迎来到图书馆，有什么需要帮助的吗？"},
      {"type": "choice", "options": [
        {"text": "我想借书", "next": "borrow_book"},
        {"text": "只是看看", "next": "just_looking"}
      ]}
    ]
  }
}
```

### 6.5 谜题系统

7 个谜题及其核心机制：

| 谜题 | 地点 | 类型 | 难度 | 夜晚限定 |
|------|------|------|------|---------|
| 桂花密语 | 桂中路 | 观察+时机 | ★☆☆ | 是 |
| 建校之钥 | 南湖综合楼 | 知识+输入 | ★★☆ | 否 |
| 书海寻踪 | 图书馆 | 问答+排列 | ★★★ | 否 |
| 地砖阵法 | 博雅广场 | 观察+顺序 | ★★☆ | 否 |
| 精准一击 | 佑铭体育馆 | 小游戏+密码 | ★★☆ | 否 |
| 饭卡迷踪 | 学子食堂 | 搜索+对话 | ★☆☆ | 否 |
| 七徽归位 | 喷泉广场 | 综合+排列 | ★★★ | 否 |

谜题状态枚举：`UNDISCOVERED` → `DISCOVERED` → `IN_PROGRESS` → `SOLVED`

### 6.6 昼夜循环

- 游戏内 1 天 = 现实 10 分钟（600 秒）
- 时段划分：白天(6:00-18:00) → 黄昏(18:00-19:30) → 夜晚(19:30-6:00)
- 光照实现：全局色调叠加渐变
  - 白天：无叠加
  - 黄昏：半透明橘红色层
  - 夜晚：半透明深蓝色层 + 路灯/窗户发光
  - 秘境异变：半透明幽绿色层

### 6.7 体力系统

- 最大体力：100
- 冲刺消耗：1 点/秒
- 站立恢复：0.5 点/秒
- 体力耗尽：无法冲刺，移动速度降为 0.7 倍

### 6.8 存档系统

- JSON 序列化存档
- 1 个自动存档槽 + 2 个手动存档槽
- 自动存档触发点：退出游戏、进/出室内、跨校区、解谜、拾取关键道具
- 存档时写临时文件再重命名，避免写入中断导致数据损坏

### 6.9 地图过渡

| 过渡类型 | 实现方式 | 过渡效果 |
|----------|---------|---------|
| 室外→室内 | 走到门口按F | 淡出淡入（0.5秒） |
| 室内→室外 | 走到出口自动触发 | 淡出淡入（0.5秒） |
| 楼层切换 | 走到楼梯按F | 淡出淡入（0.5秒） |
| 跨校区 | 校区接驳站按F | 校车过场动画（2-3秒）+ 淡入 |

## 七、UI 设计规范

### 7.1 UI 技术架构

采用三层架构：

1. **pygame_gui**：负责交互逻辑（布局、点击、悬停、焦点、滚动、窗口管理）
2. **自定义渲染层**：负责像素风外观（9-slice 边框、像素图标、动画效果）
3. **自定义主题 (theme.json)**：像素风字体、颜色/间距配置、按钮图片路径

### 7.2 UI 三阶段实现路径

| 阶段 | 对应日程 | 内容 |
|------|---------|------|
| 阶段1 | Day 1-11 | 基础功能 UI，pygame_gui 纯色边框，矩形占位 |
| 阶段2 | Day 12-13 | 素材替换，9-slice 分层渲染，Zpix 字体，道具图标 |
| 阶段3 | Day 14 | 精细打磨，体力条桂花主题，小地图羊皮纸质感，动画效果 |

### 7.3 HUD 布局

### 7.4 9-slice 边框技术

阶段1先用 pygame_gui 纯色边框，阶段2切换为分层渲染+9-slice：
1. pygame_gui 元素设置 `border_width=0`、`shadow_width=0`，背景色设为透明
2. 在 pygame_gui 渲染之前，用自写的 9-slice 渲染函数在目标位置绘制像素风边框背景
3. pygame_gui 只负责绘制文字和交互状态

## 十二、开发策略

### 12.1 核心开发策略

1. **自底向上 + 增量迭代**：先搭建底层框架，再逐层叠加功能，每一步都可运行测试
2. **先跑通再打磨**：每个模块先实现最小可用版本，再逐步完善细节
3. **数据驱动**：对话、道具、谜题数据全部外置为 JSON，减少硬编码
4. **UI 三阶段递进**：先纯色/矩形占位，再替换像素风素材，最后精细打磨

### 12.2 降级方案优先级

如果进度落后，按以下优先级依次砍功能：

1. **P2 可砍**：音效音乐
2. **P1 可简化**：昼夜循环 → 简化为固定白天
3. **P1 可简化**：小地图 → 简化为静态图片
4. **P1 可简化**：道具组合 → 取消组合功能
5. **P1 可简化**：结局差异 → 仅实现普通结局
6. **P0 不可砍**：7 个谜题 + 通关流程 + 存档

---

## 十三、性能优化规范

1. **视口裁剪**：仅渲染摄像机可见范围内的 Tile 和实体
2. **脏标记渲染**：静态地形仅在变化时重绘，动态实体每帧更新
3. **实体剔除**：远离玩家的 NPC/物体不更新逻辑
4. **资源预加载**：进入区域时预加载相关资源
5. **地图分块**：大地图按 Chunk 分块加载，按需载入/卸载

---

## 十四、安全与健壮性规范

1. 存档写入使用"写临时文件→重命名"策略，防止写入中断导致数据损坏
2. JSON 数据文件加载时须做异常捕获，缺失字段使用合理默认值
3. 地图加载失败时应有降级处理（如显示错误提示而非崩溃）
4. 禁止在代码中暴露或打印敏感信息
